# TP 28 — SonarQube (Local)
## Analyse d’un projet Java Maven, Quality Gate et Issues
# Par: Rim EL ABBASSI

## Objectif de l’activité

L’objectif de ce TP est de mettre en place **SonarQube en local** à l’aide de **Docker**, puis de réaliser l’analyse statique d’un projet **Java Maven**.  
À la fin de ce travail, il sera possible de :
- Démarrer et configurer SonarQube localement
- Créer un projet SonarQube en mode manuel
- Générer et utiliser un token d’authentification
- Lancer une analyse Maven avec SonarScanner
- Interpréter les résultats (Quality Gate, bugs, vulnérabilités, code smells, couverture)
